# 实施计划（面向 AI 开发者，分步指令）

## 范围与节奏
先完成基础功能（MVP），再扩展完整功能。每一步都包含验证正确性的测试。MVP 以可演示为目标，前端需兼顾基本美观与可用性。

---

## 一、基础功能（MVP）

### 步骤 1：确认需求与最小范围
指令：
- 从 `design-document.md` 与 `tech-stack.md` 提炼“基础功能清单”，明确仅包含：用户登录、课程/知识库上传、RAG 问答、练习生成、练习评测、基础统计。
- 列出不做项（如大屏、复杂学情分析），确保单人可交付；三角色页面保留但功能最小化。
- 明确基础统计口径仅为：用户数、练习次数、正确率。
前端开发：
- 无页面开发，仅同步需求与范围到前端需求清单。
后端开发：
- 无接口开发，仅同步需求与范围到接口清单。
测试：
- 输出一份“基础功能清单+不做项”文档，逐条对照原文档确认无遗漏/无越界。

### 步骤 2：搭建项目结构
指令：
- 建立前端、后端、模型服务与数据存储的最小目录结构。
- 约定 API 路径风格与资源命名规范。
前端开发：
- 初始化前端目录结构与基础入口文件，占位路由与页面目录。
后端开发：
- 初始化后端目录结构与基础入口文件，预留路由与配置文件位置。
测试：
- 检查目录结构与命名规范文档，确保可映射到“基础功能清单”中的每个能力模块。

### 步骤 3：准备本地知识库样本
指令：
- 选取一门课程的纲要与讲义作为样本知识库；允许一门课对应多份知识库文档。
- 制定文档清洗与切分规则（如段落级切分与最小字数阈值）。
前端开发：
- 无页面开发，仅确认后续上传与查看的页面字段需求。
后端开发：
- 无接口开发，仅确认后续知识库上传与解析的数据结构。
测试：
- 随机抽取 3 段知识点，人工确认切分结果语义完整且与课程主题一致。

### 步骤 4：建立知识库索引与检索流程
指令：
- 明确向量化与检索流程的输入输出格式。
- 配置基础检索策略（如 Top-k、重排序可选）。
- 兼容多种文档格式输入（至少支持 PDF/Word/Markdown 任意组合）。
前端开发：
- 知识库上传表单的字段与状态占位（课程选择、文件列表、上传进度）。
后端开发：
- 预留知识库上传与检索的 API 路由与服务层接口（不要求完整实现检索）。
测试：
- 用 3 个问题进行检索验证：每个问题必须命中至少 1 条相关知识片段。

### 步骤 5：完成 RAG 问答最小闭环
指令：
- 定义“问题 → 检索 → 生成”的输入输出协议。
- 设定生成答案的引用字段（知识点来源或文档片段 ID）。
前端开发：
- 问答页面的输入框、历史记录与引用展示占位。
后端开发：
- 问答 API 的请求/响应格式对齐 `rag-qa-spec.md`，返回引用字段。
测试：
- 用 5 个问题测试，回答中必须包含引用字段且能追溯到检索结果。

### 步骤 6：实现练习生成（基础）
指令：
- 约定题型（如单选/判断/简答）与题目数据结构。
- 基于知识库生成练习题并附标准答案与知识点来源。
- 简答题提供评分标准（关键点列表或分值点）。
前端开发：
- 练习生成表单与结果列表占位（题型、难度、数量）。
后端开发：
- 练习生成 API 的请求/响应格式对齐 `exercise-generation-spec.md`。
测试：
- 生成 10 道题，逐题检查：题干清晰、答案存在、来源可追溯。

### 步骤 7：实现练习评测（基础）
指令：
- 定义学生提交格式与评测输出格式（正确/错误/建议）。
- 实现基于标准答案的判定流程。
前端开发：
- 练习作答与提交入口占位，显示评测结果与反馈字段。
后端开发：
- 练习评测 API 的请求/响应格式对齐 `exercise-grading-spec.md`。
测试：
- 用 5 组“正确答案/错误答案”样本进行评测，判定结果符合预期。

### 步骤 8：实现基础用户与课程管理
指令：
- 支持注册与账号密码登录。
- 三角色（管理员/教师/学生）均需可用页面与最小功能：登录、课程查看；教师可创建课程。
测试：
- 用三种账号测试：可注册、可登录并访问各自的最小功能页面。

### 步骤 9：前后端联调（MVP）
指令：
- 按功能顺序联调：登录 → 上传知识库 → RAG 问答 → 练习生成 → 练习评测。
- 接入外部模型 API 服务完成问答与生成闭环，后续再考虑本地推理替换。
测试：
- 完整走通 1 条课程流程，确保每个环节有可见结果输出。

### 步骤 10：MVP 验收
指令：
- 形成“最小可用演示流程”文档与截图。
- 前端页面完成基本美观与可用性检查（布局、字号、留白、颜色一致性）。
测试：
- 使用新用户账号走通流程，确保无手工干预。

---

## 二、完整功能（扩展）

### 步骤 11：扩展学情分析
指令：
- 增加错题聚合、知识点掌握统计与趋势指标。
测试：
- 用 30 条模拟答题记录验证统计指标可正确更新。

### 步骤 12：扩展教师备课能力
指令：
- 自动生成课程讲解结构与时间分配建议。
测试：
- 随机抽取 2 章课程，检查生成内容结构完整且时间分配合理。

### 步骤 13：扩展管理端统计大屏
指令：
- 增加教师/学生活跃统计、教学效率指数、学习效果趋势。
测试：
- 用模拟数据验证各指标图表可正确展示。

### 步骤 14：完善资源管理
指令：
- 支持课件与练习导出（如按课程打包）。
测试：
- 导出 1 门课程资源并核对文件数量与内容完整性。

### 步骤 15：完善系统稳定性
指令：
- 增加错误提示、失败重试与日志追踪。
测试：
- 模拟 3 类异常（无知识库/模型不可用/提交格式错误），确保系统给出可理解的反馈。

### 步骤 16：最终验收
指令：
- 形成完整演示脚本与答辩材料要点。
测试：
- 使用不熟悉流程的同学试跑演示，记录问题并修复。
