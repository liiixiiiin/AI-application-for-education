# 设计文档：基于AI大模型的教学实训智能辅助系统

## 1. 项目概述
本项目为本科毕业设计，题目为"基于AI大模型的教学实训智能辅助系统的设计与实现"。面向高校实训教学场景，以开源中文大模型为核心引擎，结合 RAG（检索增强生成）技术与本地课程知识库，建设智能化教学平台原型，覆盖教师备课、学生练习与管理分析三端需求，提升教学资源利用效率与学习质量。

## 2. 业务背景
高校实训教学面临多方面痛点：教师备课与出题耗时长，内容更新滞后；学生难以及时获得个性化指导；教学数据分散，缺乏统一的分析反馈机制。将开源大模型与 RAG 技术及本地课程知识库相结合，可实现精准知识问答、内容智能生成、智能出题与自动评分等功能，有效减轻教师负担，提升学生自主学习效率，并为教学管理提供数据支持。

## 3. 目标与范围
- **目标**：构建可落地的智能实训平台原型，完成教学内容生成、在线练习、学情分析、资源管理等核心能力。
- **范围**：单校级部署，优先采用本地知识库与开源/API 模型。
- **约束**：单人可完成的工程规模，模块化实现，优先简化部署与运维。

## 4. 需求分析

### 4.1 角色与使用场景
- **教师**：上传课程大纲与知识库，自动生成教学内容与考核题；查看学情分析与教学建议。
- **学生**：在线提问获得知识讲解；生成随练题并即时纠错；基于薄弱知识点获得个性化练习推荐。
- **管理员**：用户与资源管理；教学与学习效果统计大屏。

### 4.2 功能需求

**教师侧**
- 备课与设计：根据课程大纲/知识库自动生成教学内容（讲解、实训、时间分布）。
- 考核内容生成：自动生成多样题型与参考答案（单选题、判断题、填空题、简答题），支持按知识点与难度定向出题，采用 Few-shot Prompting 控制生成质量。
- 学情数据分析：自动检测学生答案，定位错误并给出修正建议；输出班级层面的掌握情况与教学建议。

**学生侧**
- 在线学习助手：基于 RAG 的多轮对话式课程知识问答，支持流式输出与对话历史记忆。
- 实时练习评测助手：根据历史练习与要求生成随练题并即时评分纠错。
- 个性化练习推荐：基于知识追踪技术识别薄弱知识点，动态推送针对性练习，形成"练习→评估→推荐"闭环。

**管理侧**
- 用户管理：管理员/教师/学生基本管理。
- 课件资源管理：按学科管理与导出课件、练习等资源。
- 数据看板：练习参与人数、正确率、知识点掌握分布等统计信息。

### 4.3 非功能需求
- 采用开源或可本地部署的模型作为核心组件，优先中文效果好的模型。
- 知识库优先本地存储；规模控制在可单机运行范围内。
- 生成内容与知识库有明确关联，题目与答案标注知识点来源与准确性说明。

## 5. 系统设计

### 5.1 总体架构
1. **数据层**：本地知识库（课程大纲、教材、讲义、题库等）+ SQLite 业务数据库。
2. **模型层**：开源大模型 + 检索增强生成（RAG）+ 知识追踪。
3. **业务层**：备课生成、题目生成、答疑、评测、学情分析、个性化推荐、资源管理。
4. **展示层**：教师端、学生端、管理端（Vue 3 + Element Plus）。

### 5.2 关键模块说明
- **知识库管理**：文档清洗、切分（规则/大模型辅助）、向量化与混合检索（向量 + BM25 + 重排序）。
- **RAG 问答**：基于 LangChain 编排的检索→上下文→生成流水线，支持 SSE 流式输出、多轮对话记忆管理。
- **智能出题**：结构化输出控制题目格式，Few-shot Prompting 控制质量与难度，支持单选/判断/填空/简答多题型。
- **自动评分**：客观题规则匹配；主观题基于语义相似度 + 大模型判分加权融合，支持多要点拆分评分。
- **知识追踪与个性化推荐**：基于指数移动平均（EMA）方法动态估计知识点掌握度，识别薄弱知识点并驱动大模型生成针对性练习。
- **学情分析**：错题聚合、知识点掌握统计与趋势指标。

### 5.3 工业级架构升级（阶段四主线，第 10-13 周）
为提升工程化深度与对齐工业界最佳实践，系统在中期检查后进入阶段四工业级冲刺，于答辩前完成以下结构性升级：

- **微服务架构**：业务层（SpringBoot 3 + MyBatis + MySQL）与 AI 推理层（FastAPI）解耦；统一网关路由；详见 [microservice-spec.md](memory-bank/microservice-spec.md)。
- **Agent 编排**：基于 LangGraph 的状态机式 Agent，统一承载意图识别、任务拆解、工具调用、反思纠错；详见 [agent-spec.md](memory-bank/agent-spec.md)。
- **工具与技能双层暴露**：MCP Server 暴露 5 个原子 Tool（水龙头），Agent Skills 封装 3 个高层工作流（菜谱）；详见 [mcp-server-spec.md](memory-bank/mcp-server-spec.md) 与 [skills-spec.md](memory-bank/skills-spec.md)。
- **可观测性与评测体系**：LangFuse 全链路 Trace + RAGAS + LLM-as-Judge + Golden Set 自动回归 + GitHub Actions CI 阻断；详见 [evaluation-spec.md](memory-bank/evaluation-spec.md)。
- **中间件层**：Redis 提供语义缓存 / 异步任务队列 / 令牌桶限流 / 降级策略；详见 [redis-spec.md](memory-bank/redis-spec.md)。
- **分层记忆**：短期滑动窗口 + 长期摘要压缩 + 向量化召回；详见 [memory-spec.md](memory-bank/memory-spec.md)。

## 6. 数据与模型
- **模型选择**：当前通过 DashScope API 接入通义千问系列模型；可选本地部署 Qwen2.5 / DeepSeek 等。
- **Embedding 模型**：DashScope text-embedding-v3（中文优化）。
- **Rerank 模型**：DashScope gte-rerank（跨编码器重排序）。
- **知识库规模**：控制在单机可稳定运行范围内。
- **关联性保障**：每次生成内容附"知识点来源"与"关联说明"字段，便于追溯与评估。

## 7. 核心技术原理概要

### 7.1 RAG 检索增强生成
文档解析与分块 → 文本向量化 → 向量存储与检索（ChromaDB，HNSW 索引）→ 检索结果重排序（Cross-Encoder）→ Prompt 组装与大模型生成。混合检索策略：BM25 关键词检索 + 向量语义检索，综合得分排序后进行 Rerank。

### 7.2 智能出题与自动评分
- 结构化输出：Prompt 中嵌入 JSON Schema 模板约束生成格式。
- Few-shot Prompting：预置不同难度示例引导生成质量。
- 主观题评分：语义相似度基础分 + 大模型综合判分加权融合；多要点拆分评分提升可解释性。

### 7.3 知识追踪与个性化推荐
- 采用 EMA（指数移动平均）方法动态更新知识点掌握度。
- 掌握度低于阈值的知识点标记为薄弱项，优先推送相关练习。
- 技术链路：知识追踪 → 状态量化 → 提示工程 → 大模型生成个性化练习。

### 7.4 多轮对话与分层记忆
- **短期记忆**：方案 C（自动进入最近对话 + 可选新建）+ 滑动窗口（最近 10 条注入 Prompt）。
- **长期记忆**：超阈值消息触发 LLM 摘要压缩 → 向量化入长期记忆库 → 按需召回。
- 详见 [memory-spec.md](memory-bank/memory-spec.md) 与 [rag-qa-spec.md](memory-bank/rag-qa-spec.md)。

### 7.5 Function Calling 与工具编排
- LLM 通过 Function Calling 协议（DashScope 原生支持）调用工具，避免脆弱 JSON Prompt。
- 工具能力按 MCP（Model Context Protocol）暴露，可被任意 MCP 客户端复用。
- 高层任务（备课 / 个性化练习 / 简答评分）由 Agent Skills 描述工作流，按步骤组合调用 MCP Tool。

### 7.6 Agent 状态机编排
- 基于 LangGraph 实现：意图识别 → 任务拆解 → 工具调用 → 反思纠错 → 结果聚合。
- 反思节点对输出进行多维度自检（JSON 格式、知识点匹配、答案 grounded、覆盖率），不通过则回到 planner 重做。
- 详见 [agent-spec.md](memory-bank/agent-spec.md)。

### 7.7 评测与可观测性
- 三层评测：RAGAS 自动化指标 + LLM-as-Judge（评委 Qwen-Max）+ Golden Set 回归。
- LangFuse 全链路 Trace（Agent 节点 / 工具调用 / LLM 调用三层）。
- GitHub Actions CI：评测脚本失败阻断 PR。
- 详见 [evaluation-spec.md](memory-bank/evaluation-spec.md)。

## 8. 运行流程
1. 管理员初始化课程与用户。
2. 教师上传课程大纲与知识库文档。
3. 系统自动生成教学内容与考核题。
4. 学生在线学习与练习，系统即时评测并推荐个性化练习。
5. 教师与管理端查看学情分析与统计数据。

## 9. 研究难点与对策

### 9.1 算法与模型层
- **RAG 检索质量与幻觉抑制**：混合检索（BM25 + 向量）+ Cross-Encoder 重排序 + 知识库约束生成 + 输出引用。
- **出题质量可控性**：结构化输出 / Function Calling + 知识范围约束 + 反思纠错 + Golden Set 回归。
- **主观题评分准确性**：语义相似度 + 大模型多维度评分 + 评分细则嵌入 Prompt。
- **个性化推荐冷启动**：初始诊断练习快速建立能力画像。

### 9.2 工业化落地难点（阶段四主线）
- **Agent 反思纠错收敛性**：反思无限循环风险，需重试上限与降级策略（详见 [agent-spec.md](memory-bank/agent-spec.md) §8）。
- **缓存一致性**：知识库变更后语义缓存如何及时失效（用 `kb_version` 信号，详见 [redis-spec.md](memory-bank/redis-spec.md) §3）。
- **双服务鉴权透传**：JWT 跨服务有效性验证、密钥同步（详见 [microservice-spec.md](memory-bank/microservice-spec.md) §5.2）。
- **评测体系成本控制**：LLM-as-Judge 高频运行的 token 成本与延迟（用 Qwen-Max 而非 GPT-4，定期跑而非每次跑）。
- **长期记忆隐私**：摘要 Prompt 强制去标识化，跨用户严格隔离（详见 [memory-spec.md](memory-bank/memory-spec.md) §7）。

## 10. 验收与评估指标
- **功能性指标**：备课效率提升（时间节省比例）、题目生成准确性、学生练习正确率提升趋势、高频错误知识点收敛。
- **RAG 指标**：faithfulness / answer_relevancy / context_precision / context_recall。
- **Agent 指标**：意图识别命中率、任务成功率、反思触发率、平均工具调用数。
- **工程指标**：p95 延迟、缓存命中率、限流触发数、CI 评测通过率。
- 详细指标体系见 [evaluation-spec.md](memory-bank/evaluation-spec.md)。
