# 设计文档：高校实训智能体教学平台

## 1. 项目概述
本项目为单人毕业设计，面向高校实训教学场景，建设智能化教学平台原型，覆盖教师备课、学生练习与管理分析三端需求，提升教学资源利用效率与学习质量。赛题内容仅作为参考，功能与约束以毕设可落地性为优先。

## 2. 赛题业务场景（企业背景简化）
某大型职业教育集团下辖多所高校与实训基地，涉及计算机、机电、商科等多个专业。教师需频繁更新实训内容并针对不同班级制定题目，人工备课耗时长且资源重复利用率低；学生课后缺少及时指导，练习纠错依赖教师批改，反馈慢。集团希望引入开源大模型与本地知识库，构建“实训智能体”，实现教学内容自动生成、在线练习评测、错误定位与学习资源推荐，并在管理端形成教学效率与学习效果的数据闭环。为适配比赛，场景简化为单校多专业、单平台三端协作。

## 3. 目标与范围
- 目标：构建可落地的智能实训平台原型，完成教学内容生成、在线练习、学情分析、资源管理等核心能力。
- 范围：本项目聚焦单校级部署，优先采用本地知识库与开源模型；如需更好效果，可在条件允许下接入外部模型 API。
- 约束：单人可完成的工程规模，模块化实现，优先简化部署与运维。

## 4. 需求分析
### 4.1 角色与使用场景
- 教师：上传课程大纲与知识库，自动生成教学内容与考核题；查看学情分析与教学建议。
- 学生：在线提问获得知识讲解；生成随练题并即时纠错。
- 管理员：用户与资源管理；教学与学习效果统计大屏。

### 4.2 功能需求
**教师侧**
- 备课与设计：根据课程大纲/知识库自动生成教学内容（讲解、实训、时间分布）。
- 考核内容生成：自动生成多样题型与参考答案（如编程题、判断题、简答题）。
- 学情数据分析：自动检测学生答案，定位错误并给出修正建议；输出班级层面的掌握情况与教学建议。

**学生侧**
- 在线学习助手：基于教学内容答疑。
- 实时练习评测助手：根据历史练习与要求生成随练题并纠错。

**管理侧**
- 用户管理：管理员/教师/学生基本管理。
- 课件资源管理：按学科管理与导出课件、练习等资源。
- 大屏概览：统计教师/学生活跃、教学效率指数、学生学习效果趋势与高频错误知识点。

### 4.3 非功能需求
- 采用开源或可本地部署的模型作为核心组件，优先中文效果好的模型。
- 知识库优先本地存储；规模控制在可单机运行范围内即可。
- 生成内容与知识库有明确关联，题目与答案建议标注知识点来源与准确性说明。

## 5. 系统设计
### 5.1 总体架构
1. **数据层**：本地知识库（课程大纲、教材、讲义、题库等）。
2. **模型层**：开源大模型 + 检索增强生成（RAG）。
3. **业务层**：备课生成、题目生成、答疑、评测、学情分析、资源管理。
4. **展示层**：教师端、学生端、管理端。

### 5.2 关键模块说明
- 知识库管理：文档清洗、切分、向量化与检索。
- 教学内容生成：基于课程大纲与知识库生成结构化教学方案。
- 考核题生成：按学科题型模板生成题目、答案与知识点映射。
- 评测与纠错：自动比对标准答案，给出错误定位与改进建议。
- 学情分析：统计正确率、知识点掌握、常见错误及教学建议。

## 6. 数据与模型
- 模型选择：优先国产或中文友好模型，如 Qwen2.5 / DeepSeek / Yi 等；可本地部署或本地 API 服务。
- 知识库规模：控制在单机可稳定运行范围内。
- 关联性保障：每次生成内容附“知识点来源”与“关联说明”字段，便于追溯与评估。

## 7. 实现条件（单人毕设可落地配置）
- 开发环境：macOS 或 Windows，Python 虚拟环境 + Node.js；若条件允许可接入本地模型 API 服务。
- 实验平台：单机本地部署（CPU 或 GPU 皆可），可选接入外部模型 API 用于对比实验。
- 开发语言：后端 Python，前端 TypeScript/JavaScript。
- 数据库：SQLite（单机），可选 PostgreSQL 作为升级方案。
- 编译器/运行时：Python 3.10+，Node.js 18+；前端构建使用 Vite。
- 涉及硬件：普通笔记本可运行，建议 16GB 内存；如有 GPU 更佳。

## 8. 运行流程
1. 管理员初始化课程与用户。
2. 教师上传课程大纲与知识库文档。
3. 系统自动生成教学内容与考核题。
4. 学生在线学习与练习，系统即时评测。
5. 教师与管理端查看学情分析与统计数据。

## 9. 验收与评估指标
- 备课效率提升（时间节省比例）。
- 题目生成准确性与知识点覆盖率。
- 学生练习正确率提升趋势。
- 学生高频错误知识点的收敛情况。

## 10. 风险与对策
- 模型幻觉：使用知识库检索约束 + 输出知识点来源。
- 知识库质量不均：文档清洗与分级管理。
- 评测偏差：结合规则校验与教师复核机制。

## 11. 里程碑计划（简要）
- 第1周：需求梳理与架构设计
- 第2-3周：知识库与RAG原型搭建
- 第4-5周：教师/学生/管理端核心功能
- 第6周：测试与优化、文档整理
